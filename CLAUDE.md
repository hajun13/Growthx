# GrowthX — 인사평가 솔루션

## 하네스: 인사평가 솔루션 풀스택 개발

**목표:** 인사평가(HR 성과평가) 솔루션 웹사이트를 와이어프레임부터 Docker 배포까지 5인 전문 에이전트(디자인·프론트·백엔드·QA·릴리스)로 개발한다. **기본 실행 = 필요한 에이전트만 띄우는 최소 범위 서브에이전트**(토큰·속도 절감), 5인 풀팀은 신규 대형 빌드에서만. 모델은 backend만 opus, 나머지 sonnet (상위 모델 일괄 상향 금지 — 속도는 병렬 팬아웃으로). 프론트 작업은 `nextjs-frontend` 스킬 필수.

**트리거:** 인사평가/성과평가/평가 시스템·솔루션 관련 개발 작업(화면 설계, 프론트엔드, 백엔드 API, 통합 QA, 배포)이나 그 부분 재실행·수정·보완 요청 시 `eval-harness-orchestrator` 스킬을 사용하라. 단순 질문은 직접 응답 가능.

**제품명:** **에너지엑스 인사 평가** (이전 "GrowthX" 브랜드 폐기 — UI/문서 표기는 모두 "에너지엑스 인사 평가").

**스택:** Next.js(App Router) 프론트 + **Part 하이브리드 디자인 시스템**(루트 `DESIGN.md`가 디자인 SSOT — 2026-07-03 전면 재작성), **기본 글꼴 Pretendard**(한글 최적화) + NestJS/Prisma/PostgreSQL **모듈러 모놀리식** 백엔드 + **OpenAPI 계약우선**(`@nestjs/swagger` 발행 → orval codegen) + Docker 자체 호스팅. **Turborepo + pnpm 모노레포**(`apps/*` 배포 단위 + `packages/*` 공유: ui·contracts·auth·config) — **서비스가 늘어나는 멀티서비스 플랫폼** 전제. 등급·풀·인상률·가중치 등 수치 규칙은 **설정 가능(`RuleSet`)**, 에너지엑스 2026 값을 기본 seed. **E2E 테스트는 루트 `e2e/`**(Playwright Test Agents — 🎭planner→generator→healer, `pnpm test:e2e`; 인증은 localStorage `gx.*` storageState 주입, 앱은 `webServer` 자동기동/`reuseExistingServer`). Playwright 업데이트 시 `npx playwright init-agents --loop=claude` 재생성.

> **아키텍처 SSOT(2026-06-15):** 코드 구조의 권위는 `eval-harness-orchestrator/references/architecture.md`. 모노레포 레이아웃·모듈 경계(미래의 절단선·DB schema 분리)·수직슬라이스·**파일당 ~200줄 상한**·모듈 매니페스트 README를 정의(바이브 코딩 — AI 가독성). 외부 API 소비/제공은 `integration-adapter.md`(부패 방지 어댑터). **하네스 문서·에이전트·스킬은 정렬 완료, 코드 모노레포화는 Phase 2~3에서 점진 적용**(현재 `apps/web`+`apps/api`는 그대로 한 서비스).

**핵심 도메인(권위 자료 PPT·xlsx 확정):** 평가 = **KPI/성과 중심**(연봉·최종등급 산정은 실적만). **역량평가는 폐기 아님 — 참고용 백데이터로 존재**(연 1회 12월·10문항·S/A/B/C/D, 가중치 100점이나 **연봉·등급 미반영**, `CompetencyQuestion`/`CompetencyResponse`로 구현, 조회·연도비교 화면에 표시). 평가 유형 = **본인평가(self) + 다단계 부서장 평가(downward 3단계: round1 팀장·round2 본부장·round3 그룹대표, 단계가중 0.5/0.3/0.2 + 평가자 동일인 예외)** — 수평/상향/다면평가 **없음**. 조직 = **그룹→본부→팀→개인 4단계**(그룹 최상위). 등급 풀 단위 = **그룹**. KPI 2그룹 = 성과중심(매출액·공정액·수주&업무수행 70/80%) + 협업·성장(협업성과·자기개발 20/30%). 등급은 측정방식별(금액 달성률/건수).
