# EnergyX Notion-Low-Color Design System

에너지엑스 인사 평가의 시각 언어 SSOT. 이 버전은 Notion처럼 따뜻한 문서형 업무 도구를 목표로 한다. 앱은 웜 오프화이트 캔버스, 흰 표면, 헤어라인, 흑백 타이포, 단 하나의 액션 블루로 구성한다. 색은 업무 판단을 돕는 최소한의 신호로만 쓴다.

## 1. 원칙

- **저채도 우선:** 구조 색은 흑백/웜 그레이가 기본이다.
- **액션 컬러 1개:** primary는 Notion Blue `#0075DE` 하나만 사용한다.
- **카드 무라운드:** 카드, 패널, 표 컨테이너, 섹션 박스는 `border-radius: 0`.
- **컨트롤 최소 라운드:** 버튼, 입력, 셀렉트, 작은 칩은 사용성을 위해 3-4px만 허용한다.
- **Pretendard 유지:** 한국어 본문 가독성을 위해 행간은 1.55-1.65, 자간은 0 또는 아주 약한 음수만 쓴다.
- **AI 느낌 제거:** 과한 그라데이션, 글래스, 컬러 그림자, 장식용 컬러 블록, 둥근 카드 묶음을 금지한다.
- **업무 밀도:** 빈 여백은 실제 상태, 다음 작업, 검증 요약, 최근 변경, 처리 큐 같은 운영 정보로 채운다.

### 제한적 상태색

상태색은 작은 점, 아이콘, 텍스트, 얇은 좌측 보더에만 쓴다. 큰 배경 면이나 카드 채움으로 쓰지 않는다.

| 상태 | 값 | 용도 |
| --- | --- | --- |
| success | `#168A45` | 완료/정상 텍스트 |
| warning | `#A66A00` | 주의 텍스트 |
| danger | `#C23A3A` | 오류/위험 텍스트 |
| info | `#0075DE` | 정보/진행 텍스트 |

### 등급 색

등급 색은 차트와 작은 등급 배지에만 제한한다. 표면 배경은 기본적으로 흰색 또는 웜 그레이를 유지한다.

| 등급 | 색 |
| --- | --- |
| S | `#111111` |
| A | `#2F2E2C` |
| B | `#615D59` |
| C | `#8A8178` |
| D | `#C23A3A` |

## 3. 타이포그래피

- 단일 패밀리: `"Pretendard Variable", Pretendard, -apple-system, "Apple SD Gothic Neo", "Malgun Gothic", system-ui, sans-serif`
- 페이지 제목: 24-28px, 700, line-height 1.35
- 섹션 제목: 16-18px, 700, line-height 1.45
- 본문: 14-15px, 400-500, line-height 1.6
- 표/메타: 12-13px, 500-600, line-height 1.45
- 숫자: `font-variant-numeric: tabular-nums`
- 긴 한국어 문장은 `break-keep`을 기본으로 하되 좁은 폭에서는 줄바꿈을 허용한다.

## 4. 간격과 레이아웃

- 기준 단위: 4px.
- 카드/패널 내부: 16-24px.
- 화면 상하좌우 여백: desktop 24-32px, mobile 16px.
- 섹션 간격: 20-28px.
- 대시보드/관리 화면은 `주요 작업 영역 + 보조 정보 레일` 또는 `필터바 + 표 + 우측 세부 패널` 패턴을 우선한다.
- 빈 공간은 장식으로 채우지 않고 실제 업무 데이터, 다음 행동, 검증 상태, 최근 변경 이력으로 채운다.

## 5. 반경

| 대상 | 값 |
| --- | --- |
| 카드, 패널, 표 컨테이너, 섹션 박스 | `0px` |
| 버튼, 입력, 셀렉트, 텍스트영역 | `4px` |
| 작은 배지, 상태칩 | `3px` |
| 아바타, 진행 점, 원형 아이콘 | `9999px` |

큰 반경 유틸리티와 카드형 pill UI는 신규 코드에서 금지한다.

## 6. 엘레베이션

- 기본은 보더만 사용한다.
- 카드 그림자 없음.
- 드롭다운/모달만 낮은 중립 그림자를 허용한다.
- 컬러 섀도 금지.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--elevation-0` | `none` | 카드/패널 |
| `--elevation-1` | `0 1px 2px rgba(17,17,17,.06)` | 드롭다운 |
| `--elevation-2` | `0 10px 28px rgba(17,17,17,.14)` | 모달 |
| `--focus-ring` | `0 0 0 2px rgba(0,117,222,.22)` | 키보드 포커스 |

## 7. 컴포넌트 규칙

- **Button:** primary만 파란 배경. secondary는 흰 배경+헤어라인. ghost는 텍스트형.
- **Card/Panel:** 흰 배경, 1px 헤어라인, radius 0, shadow none.
- **Table:** 헤더는 웜 그레이, 행 구분선은 헤어라인. 컬러 배경 행 금지.
- **Badge/Chip:** 작은 사각 라벨. 색 배경 대신 보더/텍스트 중심.
- **Tabs:** 하단 1px 또는 2px 인디케이터만 사용.
- **Modal/Popover:** radius 0-4px, 흰 표면, 낮은 중립 섀도.
- **Charts:** 흑백/그레이 우선, 비교 대상 하나만 primary blue.
- **Icons:** Lucide line icon, `currentColor`, 16-20px. 이모지 금지.

## 8. 페이지 패턴

- **업무 홈/대시보드:** 오늘 할 일, 마감, 평가 단계, 미완료 큐, 최근 변경을 한 화면에 배치한다.
- **작성/평가 폼:** 좌측 작성 영역, 우측 기준/상태/검증 레일.
- **관리 화면:** 상단 필터바, 중앙 테이블, 우측 세부 패널.
- **결과/리포트:** 수치와 표를 우선하고, 색 대신 굵기·정렬·헤어라인으로 위계를 만든다.
- **빈 상태:** 안내 문장 + 실제 다음 행동 버튼. 장식 일러스트 대신 간결한 아이콘만 사용한다.

## 9. 금지 목록

- 퍼플 중심 팔레트, 장식 컬러 블록, 대형 상태색 배경.
- 그라데이션, 글래스모피즘, 컬러 섀도.
- 둥근 카드, 카드 안의 카드 남발.
- 의미 없는 마케팅형 히어로.
- 데이터가 필요한 화면의 빈 장식 공간.
