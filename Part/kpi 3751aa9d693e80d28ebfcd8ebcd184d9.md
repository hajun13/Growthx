# **로그인 화면**

현재 로그인 화면은 기능 중심으로 구성되어 있으나, 전반적인 가독성과 사용자 경험(UX)이 부족하여 디자인을 전면 개선 필요

- 디자인 방향
    - 심플하고 세련된 SaaS 플랫폼 스타일 적용
    - 불필요한 기능 노출 최소화
    - 사용자 친화적인 UI 구성
    - ENERGYX 브랜드 아이덴티티 반영
    - 색 통일
        - 메인 : 왼쪽 카테고리 R:86 / G:69 / B:153
        - 블루 : R:2 / G:87 / B:206
        - 녹색 : R:14 / G:208 / B:217
        - 그레이 : R:248 / G:249 / B:253
- 향후 화면 개발 방향
    
    이번 로그인 화면 디자인을 기준으로 향후 개발되는
    **모든 화면(대시보드, KPI 작성, 평가, 관리자, 목표보드 등)의 디자인 시스템을 동일하게 적용**
    
    즉, **로그인 화면을 KPI 플랫폼의 디자인 기준(Design Guide)으로 활용하여 전체 화면의 통일**

# 사용자 관리

- **[사용자 관리 화면 개선 요청]**
    
    사용자 관리 목록의 아래 항목에 정렬(Sort) 기능 추가 요청
    
    - 그룹/본부
    - 팀
    - 직급
    - 입사일
    - 상태
    
    컬럼명을 클릭하면 오름차순/내림차순 정렬이 가능하도록 적용 부탁드립니다.
    
    추가로 향후 인원 증가에 대비하여 그룹, 팀, 직급 기준으로 사용자를 빠르게 조회할 수 있도록 정렬 기능을 제공
    
    현재는 사용자가 많아질수록 원하는 대상을 찾는데 시간이 소요되어 관리 효율성이 떨어질 수 있습니다.
    
    검색 기능과 함께 사용할 수 있도록 구현 부탁드립니다.
    
    ![image.png](image.png)
    
- **[조직 구조 화면 개선 요청]**
    
    조직 구조 화면에서 아래 내용 개선 요청
    
    1. 구성원 추가 기능 추가
    
    - 현재 선택한 본부/팀에 **구성원을 추가할 수 있는 기능이 없음**
    - **“구성원 추가” 버튼을 추가**하여 선택한 조직에 직원을 바로 배치할 수 있도록 부탁드립니다.
    
    2. 레이아웃 정렬 개선
    
    - 상단 “조직 구조” 영역과 우측 “정리해야 할 항목” 박스의 크기와 정렬이 맞지 않음
        - 조직구조하고 정리해야 할 항목
    - 두 영역의 높이와 여백을 통일하여 화면이 정돈되어 보이도록 수정
    
    조직 관리 화면에서 구성원 추가 및 배치 업무를 빠르게 처리하기 위한 개선 요청.
    
    ![image.png](image%201.png)
    
- **[권한 관리 화면 개선 요청]**
    
    현재 권한 관리 화면에서는 권한 레벨 기준으로만 조회가 가능합니다.
    
    사용자 수 증가에 대비하여 아래 기준으로 정렬 및 필터링 기능 추가를 요청
    
    - 그룹별 조회
    - 본부별 조회
    - 팀별 조회
    
    예시)
    
    그룹 선택 → 본부 선택 → 팀 선택
    
    방식으로 단계별 조회가 가능하도록 적용 부탁
    
    현재는 특정 팀 또는 본부 사용자의 권한을 확인하려면 직접 검색해야 하므로 관리 효율이 떨어집니다.
    
    조직 단위로 권한을 관리하고 점검할 수 있도록 개선 부탁

## 중간점검

- **[중간점검 화면 개선 요청]**
    
    현재 구조는 담당자 작성 내용과 상급자 검토 영역의 구분이 명확하지 않아, 검토자가 어떤 내용을 확인하고 어떤 행동(승인/반려/수정요청)을 해야 하는지 한눈에 파악하기 어렵움
    
    아래 방향으로 개선 필요
    
    1. 담당자 작성 영역과 상급자 검토 영역 분리
    - 좌측 : 담당자 중간점검 내용
    - 우측 : 상급자 검토 의견 및 승인 프로세스
    - 작성자와 검토자의 역할을 명확히 구분
    1. 상급자 중간점검 기능 추가
    - 상급자가 중간점검 의견 작성
    - 반려
    - 수정요청
    - 승인
    기능 제공
    1. 버튼 디자인 정리
    - 승인 : 파란색 강조
    - 반려, 수정요청 : 그레이 톤
    - 주요 액션(승인)만 강조하여 시선 집중
    1. 구성원 리스트 개선
    - 이니셜(박, 김, 최 등) 대신 실제 프로필 사진 표시
    - 누가 작성한 내용인지 직관적으로 확인 가능하도록 개선
    1. 가독성 향상
    - KPI별 카드 형태 유지
    - 담당자 작성 내용 / 상급자 의견 / 승인상태를 구분하여 표시
    - 현재 디자인의 심플함은 유지하되 검토 프로세스가 명확하게 보이도록 개선
    
    목적은 단순 조회 화면이 아니라 팀장·본부장이 실제 검토 및 피드백을 진행하는 화면이라는 점을 명확하게 보여주기 위함

## **이의 제기**

- **반영필요**
    1. 좌측 이의제기 목록 개선
        - 이니셜 대신 신청자 프로필 사진 표시
        - 상태(접수/검토중/답변완료/최종완료/반려) 배지 추가
        - 제목, 신청자, 날짜가 한눈에 보이도록 카드 형태로 정리
    
    2. 상단 신청자 정보 영역 개선
    
    - 신청일, 현재 상태, 처리 담당자를 카드 형태로 요약
    - 현재 상태는 색상으로 구분
    
    3. 진행 상태 표시 개선
    
    - 접수 → 검토중 → 부서장 답변 → HR 최종결정
    - 단계별 완료일 및 담당자 표시
    - 현재 진행 단계가 명확하게 보이도록 개선
    
    4. 처리 내용 영역 재구성
    
    아래 순서로 카드형 UI 적용
    
    - 신청 내용
    - 부서장 답변
    - HR 검토
    - 첨부파일
    
    현재보다 각 영역을 명확하게 구분하여 가독성 향상
    
    5. 첨부파일 기능 강화
    
    - 계약서, 실적자료, 증빙자료 등 첨부파일 표시
    - 다운로드 가능하도록 구성
    
    6. 최종 결정 영역 개선
    
    현재 단순 사유 입력 방식에서 아래 선택 기능 추가
    
    - 평가 유지
    - 점수 수정
    - 등급 수정
    - 재평가 진행
    - 기각
    
    선택 후 최종 결정 사유 입력

    현재 화면은 이의제기 처리 흐름이 한눈에 보이지 않음. 신청 → 검토 → 답변 → 최종결정 프로세스가 자연스럽게 보이도록 개선하여 HR 및 관리자 사용성을 높이기 위함

## **분포 모니터링**

- **수정 요청**
    1. 상단 등급 카드 개선
        - 현재 단순 숫자 표시 형태 → S/A/B/C/D 등급 색상 체계 적용
            - S : 보라
            - A : 초록
            - B : 주황
            - C : 노랑
            - D : 빨강
        - 등급별 인원수, 비율을 카드 형태로 강조
        - 전체 대상자 수, 평균점수, 최고등급은 우측 상단 KPI 형태로 유지
    2. 전사 등급 분포 개선
        - 현재 흑백 막대 그래프 → 등급별 컬러 적용
        - 분포율(%)과 인원수를 함께 표시
        - 전사 분포 현황을 한눈에 확인 가능하도록 개선
    3. 필터 기능 추가
        - 그룹
        - 본부
        - 팀
        - 직급
        - 등급(S/A/B/C/D)
        - 정렬기준(점수순, 이름순 등)
    4. 부서별 분포 개선
        - 현재 단순 막대 형태 → 등급별 컬러 누적바 적용
        - 각 팀별 인원수 표시
        - 팀별 등급 편중 여부를 쉽게 확인 가능하도록 개선
    5. 결과 리스트 개선
        - 프로필 사진 추가
        - 순위 표시
        - 최종점수 강조
        - 등급 배지 색상 통일
        - "상세보기" 버튼 추가
    6. 하단 안내문 추가
        - 점수 산정 기준
        - 등급 기준
        - 평가 반영 방식
        - 분포 참고사항 안내
    7. 전체 디자인 방향
        - KPI 평가 전체 화면과 동일한 디자인 톤 적용
        - 카드형 레이아웃 사용
        - 불필요한 흑백 UI 최소화
        - 가독성 중심으로 심플하게 구성
        - S/A/B/C/D 색상 체계 전 화면 동일 적용
        
        목표는 단순 결과 조회가 아니라,
        
        대표진과 본부장이 "조직별 평가 분포와 편중 현황을 한눈에 확인"할 수 있는 관리형 대시보드 형태로 개선하는 것입니다.
