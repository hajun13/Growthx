## UX 개선 핵심

### KPI 검토 (`kpi/review`)
1. **전체 진행률 바**: 처리된 과제 / 전체 과제를 프로그레스 바로 표시, 달성 시 틸 그라데이션
2. **일괄 승인**: 현재 선택 팀원의 submitted KPI를 한번에 승인·확정, 확인 Modal 2단계 (인원명·개수 표시)
3. **요약 카드 수치 스케일**: `text-[34px] font-extrabold` 대형 수치 + 아이콘 타일로 §3-1 패턴 준수
4. **InfoBanner 상태 안내**: 검토 완료/검토 대기 없음/권한 없음 메시지를 plain text → InfoBanner로 교체
5. **반려/수정요청 모달 보강**: 사유 미입력 인라인 에러 + 설명 텍스트 구분 (반려 vs 수정요청 의미 안내)

### 이의신청 (`appeals`)
1. **상태 타임라인 4단계**: 접수 → 검토중 → 부서장 답변 → 최종 처리, 목록 카드(토글) + 상세 패널 상단 고정
2. **요약 카드 대형 수치**: `text-[34px] font-extrabold` + 카테고리 아이콘
3. **ContentBlock 구분**: ① 신청 사유 / ② 부서장 답변 / ③ HR 최종 결정을 배경색·좌측 보더로 명확히 구분
4. **폼 검증 인라인 에러**: 최소 10자 미만 시 에러 메시지 필드 하단 표시 (Toast 병행)
5. **EmptyState + 행동 버튼**: 필터별 빈 상태에서 "전체 보기" 버튼 제공

### 알림 (`notifications`)
1. **안읽음/읽음 섹션 분리**: 안읽음은 파란 배경 틴트(rgba(0,84,202,0.025)) + 섹션 헤더 배지로 명확히 구분
2. **날짜별 그룹 헤더**: 읽음 목록을 오늘/어제/N일 전/날짜 그룹으로 구분, Clock 아이콘 + 날짜 레이블
3. **필터 탭 개수 배지**: 각 탭에 해당 카테고리 알림 수 표시
4. **빈 상태 개선**: 안읽음 탭 → "모든 알림을 확인했어요.", 필터별 "전체 알림 보기" 버튼
5. **스켈레톤 레이아웃 일치**: 실제 카드 구조와 동일한 스켈레톤 패턴

## 공용 컴포넌트 미수정 기록

- **`NotificationItem`**: NotificationBell / AppShell에서도 사용 — 수정 금지. 알림 페이지는 페이지 레이아웃 레이어에서만 개선(날짜 그룹·섹션 분리).
- **`PageHeader`, `Button`, `Modal`, `States`, `InfoBanner`, `Tabs`, `Card`**: 수정 없이 props/slots만 활용.
