## 3. 구현 원칙
- 데이터 레이어(훅·API 호출)는 유지, 표현 계층 위주 수정. API에 없는 데이터를 추측 생성하지 않음(폴백 + 갭 기록).
- 공용 토큰(등급 색·Avatar)은 Foundation에서 1회 변경 — 페이지 에이전트는 공용 컴포넌트/lib 수정 금지, 자기 feature 디렉토리만.
- 파일당 ~200줄 상한(architecture.md), 초과 시 하위 컴포넌트 분리.
- 검증: tsc + next build (프리뷰 검증 금지 — 메모리 규칙).
