### 트리(OrgTree) 동작
- 계층: **회사(루트, 표시 전용) › 그룹 › 본부 › 팀**. division 없이 team이 group 직속 가능 → 그 경우 본부 레벨 건너뛰고 그룹 아래 팀 직접 표시.
- 각 노드 우측 **인원수 배지**(`Badge` secondary, 하위 포함 카운트). 직속/하위 구분이 필요하면 hover 툴팁 "직속 N · 하위포함 M".
- 펼침/접힘 chevron(`ChevronRight`/`ChevronDown`). 선택 노드는 `bg-primary/[0.06] ring-1 ring-primary/15`(사이드바 active 패턴 차용).
- 검색 입력 → 매칭 노드/사람 필터(트리는 매칭 경로만 펼침, 사람은 우측 결과로 노출). 검색어 없으면 전체 트리.
- hr_admin: 노드 hover 시 `⋯` 메뉴(`DropdownMenu`) — [이름 변경][하위 노드 추가][이동][삭제]. 트리 하단 [+ 노드 추가].

### 인물 영역
- 상단 바: **선택 경로 breadcrumb**(`Breadcrumb` 재사용) + 구성원 수 + **카드/리스트 뷰 토글**(`Tabs` 2탭 또는 세그먼트 버튼) + 필터(`Select`: 직급·재직상태).
- **카드 뷰:** `OrgPersonCard` 그리드(`grid gap-4 grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`).
- **리스트 뷰:** `ResultTable`(열: 이름·직급·소속경로·이메일·연락처·[hr_admin] 작업).
- hr_admin 카드/행: hover 시 `⋯`(`DropdownMenu`) — [수정][소속 이동][비활성/삭제].

## W3. 초기 비밀번호 강제 변경 `/onboarding/password`

```

│            │ 비밀번호를 새로 설정해 주세요            │  ← 제목 │
```

### 동작·상태
- 3개 `TextField type="password"`. 입력에 따라 체크리스트 실시간(`PasswordPolicyChecklist`) — 통과 항목 ✓(success), 미통과 ○(muted). **색+아이콘+텍스트 병기**.
- 정책: 최소 8자 · "1234" 금지 · 신규≠현재(권장) · 확인 일치. 모두 통과 전 Primary 비활성.
- 제출 → `POST /auth/change-password` → 성공 시 `mustChangePassword=false`, 랜딩(`landingPath(role)`)으로 이동 + 토스트 "비밀번호를 바꿨어요. 환영해요!".
- 실패(현재 비번 불일치): 현재 비번 `TextField error`="현재 비밀번호가 일치하지 않아요.".
- [로그아웃]만 탈출구. 브라우저 뒤로/직접 URL로 다른 화면 진입 시 가드가 다시 이 화면으로.

### 반응형
- 모바일: 카드 폭 화면 패딩(px-4), 풀폭 카드. 워드마크 상단.

### 동작
- 소속: 그룹 선택 → 본부 옵션 갱신 → 팀 옵션 갱신(연쇄). 본부·팀 "직속(없음)" 옵션 제공(group 직속 team / division 직속 인원).
- role·scope: `Select`. 기본은 자동기본 계산값 프리필 + "자동(직급 기준)" 표시. 바꾸면 수동 오버라이드로 저장.
- 추가 모드: 이메일·소속·직급 필수 → 저장 시 초기 비번 1234·mustChangePassword=true 자동(백엔드). 안내 캡션.
- 저장 → `POST /users`(추가) / `PATCH /users/:id`(수정). 토스트 + 조직도 refetch.
- 비활성화: `Modal` 확인 → `DELETE /users/:id`(soft).
