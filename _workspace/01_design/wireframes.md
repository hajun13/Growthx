# 와이어프레임 — 인사평가 솔루션 (도메인 대정정 반영 · v2)

> **권위 자료:** 운영계획 PPT + `domain-model.md` + `business-rules.md`. **시각 언어:** TDS 차용(`tds-design-language.md`).
> **참고용(advisory):** `reference-ui-screens.md` — 레이아웃 아이디어만, 그대로 베끼지 않음. **레퍼런스의 역량 탭·다면(수평/상향)·레이더·산점도는 채택하지 않는다.** 충돌 시 도메인 우선.
> **소비자:** frontend-engineer가 이 화면 인벤토리·레이아웃·상태·플로우를 그대로 구현.
> **토큰/컴포넌트:** 색·치수는 `design-tokens.md`, props·상태는 `component-spec.md` 참조. 본 문서는 화면 단위 조립.

## ⚠️ v2 도메인 대정정 (이 기준으로 전면 재작성)
1. **역량평가 삭제** — 본인평가 탭 = `성과중심 지표` / `협업·성장 지표` 2탭. 역량 S~D 라디오·레이더 차트 없음.
2. **다면평가 삭제** — 수평(peer)·상향(upward) 화면/흐름 제거. 평가 = 본인(self) + 부서장(downward 1차 팀장·2차 본부장).
3. **조직 4단계** — 그룹(group) → 본부(division) → 팀(team) → 개인. 그룹이 최상위.
4. **KPI 분류** — category(매출액/공정액/수주&업무수행/협업성과/자기개발) + group(성과중심 70/80% · 협업·성장 20/30%) + measureType(amount/rate/count/qualitative). 등급은 측정방식별.
5. **등급 풀 = 그룹 단위.**

## 도메인 문자열 (정확히 준수 — `domain-model.md`)
- 등급: `S` / `A` / `B` / `C` / `D`
- KPI 그룹(`group`): `performance_core`(성과 중심 70/80%) / `collaboration_growth`(협업·성장 20/30%)
- KPI 카테고리(`category`): `revenue`(매출액) / `construction`(공정액) / `orders`(수주&업무수행성과) / `collaboration`(협업성과) / `development`(자기개발)
- 측정방식(`measureType`): `amount`(달성금액→달성률) / `rate`(증감률) / `count`(건수) / `qualitative`(정성)
- 평가 유형(`type`): `self`(본인) / `downward`(부서장, `round` 1=팀장·2=본부장) — **peer/upward 없음**
- 평가 상태: `not_started` / `in_progress` / `submitted` / `finalized`
- KPI 상태: `draft` / `submitted` / `approved` / `confirmed` (+ `rejected` / `revision_requested` → `draft`)
- 주기 상태: `draft` / `active` / `mid_review` / `calibration` / `closed`
- 이의제기 상태: `submitted` / `under_review` / `answered` / `closed`
- 직책(`position`): `ceo`(대표이사) / `division_head`(본부장) / `team_lead`(팀장) / `chief`(책임) / `senior`(선임) / `pro`(프로)
- 역할(`role`): `hr_admin` / `division_head` / `team_lead` / `employee`

## 핵심 평가 플로우 (정보 구조)

```
[로그인] → [인사평가 메인(일정)]
                │  단계 카드 진입
                ▼
  평가준비 ─► [KPI 작성·제출] ──(employee 제출)──► [KPI 검토·승인] (team_lead/division_head)
   (Kpi: draft→submitted)                         (approved→confirmed)
                │
                ▼
  본인평가 ─► [본인평가(self)] 성과중심 탭 / 협업·성장 탭 (KPI별 실적→달성률→등급 자동)
   (Evaluation self: not_started→in_progress→submitted)
                │
                ▼
  1차 평가 ─► [부서장 평가(downward round1, 팀장)] 팀원 일괄 점수/등급 + 코멘트 필수 + 그룹 풀 분포
   (Evaluation downward r1: ... → submitted)
                │
                ▼
  2차 평가 ─► [부서장 평가(downward round2, 본부장)] 동일 + 풀 상한 강제
   (Evaluation downward r2: ... → submitted)
                │
                ▼
  결과·캘리브레이션 ─► [평가결과 상세] 종합(finalGrade·percentile) + 유형별(self/d1/d2) 비교 + 코멘트
   (calibration → Evaluation finalized → EvaluationResult)
```

**전체 도메인 플로우(SSOT):** `평가준비 → 본인평가 → 1차 팀장 평가 → 2차 본부장 평가 → 결과·캘리브레이션`. **다면(수평·상향) 단계는 존재하지 않는다.**

**역할 차이:**
- `employee`(책임/선임/프로): "내 할 일"(KPI 작성·본인평가·결과).
- `team_lead`: 추가 카드 "팀원 KPI 검토 N건"(→K2), "1차 팀장 평가 N명"(→S6 round1).
- `division_head`: 추가 카드 "2차 본부장 평가 N명"(→S6 round2), "본부 KPI 목표 설정".
- `hr_admin`: 추가 카드 "전사 진행 현황 N% / 미완료 N명"(→M2), "그룹 등급 풀 산정"(→M1). 주기 셀렉트로 다른 cycle 열람.

**상태:**
- 빈(활성 cycle 없음): "진행 중인 평가 주기가 없어요." 일러스트 + (hr_admin엔 "주기 만들기"→M5).
- 로딩: 캘린더·카드 스켈레톤.
- 에러: 본문 에러 카드 + [다시 시도].
- 권한없음: 해당 없음(전 역할 접근).

## K1. KPI 작성·제출 — `/kpi`

**목적:** 본인 개인 KPI(`Kpi`) 작성·상위 연계(`parentKpiId`)·제출(`draft→submitted`). category·group·measureType·목표값·가중치 지정.
**진입:** S1 "KPI 작성 [작성하기]" / 사이드바 "KPI 작성".
**핵심 컴포넌트:** `KpiCard`, `Select`(category·group·measureType·상위 KPI), `WeightField`(합100·정성≤30), `TextField`, `StatusBadge`, `Button`(과제 추가/삭제·제출), `Modal`(삭제 확인).

**측정방식별 입력:** `amount`/`rate`=목표 금액·증감률 + 단위, `count`=목표 건수 + 등급 임계값(KPI별 grading), `qualitative`=정성(목표 서술, isQualitative 자동 on).
**역할 차이:** 누구나 본인 KPI 작성. 상위 KPI 연계 셀렉트는 본인 부서 트리(그룹→본부→팀) 상위 KPI만 노출.

## S3. 본인평가(self) — 성과중심 / 협업·성장 탭 — `/eval/self`

**진입:** S1 "본인평가 [평가하기]" / 사이드바 "본인평가".
**핵심 컴포넌트:** `Tabs`(성과중심/협업·성장), `KpiCard`, `AchievementField`(실적→달성률→등급), `WeightField`(읽기), `ScoreCard`, `GradeChip`, `ProgressDonut`, `EvidenceUpload`, `TextField`(textarea). **GradeRadio·RadarChart 사용 안 함.**

**탭별 내용 (둘 다 KPI 카드 구조 동일):**

**상태:**
- 빈(배정 KPI 없음): "아직 확정된 KPI가 없어요. KPI 작성에서 과제를 등록·확정해 주세요." + [KPI 작성하기].
- 로딩: 3분할 스켈레톤.
- 에러(저장 실패): Toast danger + 임시저장 데이터 유지(낙관 롤백).
- 권한없음: 평가 기간 외 → "지금은 본인평가 기간이 아니에요." 빈 상태.
- 제출완료: 상단 `submitted` 배지 + 읽기전용, 우하단 "제출 완료"(비활성).

## S6. 부서장 평가(downward) — 1차 팀장·2차 본부장 — `/eval/downward`

**목적:** 평가자(team_lead 1차·division_head 2차)가 팀원 KPI 성과에 점수/등급을 일괄 부여하고 **코멘트 필수** 작성. 그룹 등급 풀 분포를 실시간으로 확인하며 풀 상한 초과 시 경고·제출 차단.
**진입:** S1 "1차 팀장 평가 N명" / "2차 본부장 평가 N명" / 사이드바 "부서장 평가".

**레이아웃:** 좌 팀원 리스트 + 우 평가 패널(상단 풀 분포). `md`↓ 세로 스택.

**round별 차이:**

- **2차 (round=2, division_head):** 본부 내 팀원 + 팀장 대상. 1차(팀장) 결과를 참고로 표시. 본부장이 최종 점수·등급 부여(확정 등급 기준). **그룹 풀 상한은 2차에서 최종 강제.**

**역할 차이:** `team_lead`만 1차, `division_head`만 2차(+1차 팀장 본인 평가). `employee`·`hr_admin`(작성 불가, 열람만) 메뉴 차단/제한.
**상태:**
- 빈(대상 0): "평가할 팀원이 없어요."
- 로딩: 분포·리스트 스켈레톤.
- 에러: Toast danger + 입력 보존.
- 권한없음: 403 "부서장 평가 권한이 없어요."
- 제출완료: `submitted` 배지 + 읽기전용.

**구성:** `ComparisonBar`는 self/downward1/downward2 3행 + 전사평균 마커. `ResultTable`은 KPI 과제별 group·measureType·달성률/건수·가중치·등급·점수. 코멘트는 1·2차 부서장 코멘트(권한 내).

> **레이더 차트·산점도 없음.** 결과는 막대 비교 + 과제 테이블 + 코멘트로만 구성한다.

**역할 차이(열람 RBAC):** `employee` 본인 결과만. `team_lead` 팀, `division_head` 본부, `hr_admin` 전체. URL `userId`가 권한 밖이면 **백엔드 403**. 인상률(보상)은 본인+hr_admin에만 노출.
**상태:**
- 미공개(`finalized` 전/cycle `closed` 전): "결과는 캘리브레이션 완료 후 공개돼요." 잠금 빈 상태.
- 빈(결과 없음): "표시할 평가 결과가 없어요."
- 로딩: 카드·바 스켈레톤. 에러: 에러 카드 [다시 시도].
- 권한없음: 403 "이 결과를 볼 권한이 없어요."
