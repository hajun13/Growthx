# 와이어프레임 — 연도 누적(YoY) 평가 비교

> 권위: `requirements-yoy.md` (orchestrator) + `domain-model.md` + `business-rules.md`. 시각 SSOT: 루트 `DESIGN.md`(Toss — Primary `#3182f6`, Dark `#191f28`, border-radius **0**, 컴팩트 고밀도, Pretendard).
> 기존 자산 재사용 원칙: `apps/web/components/`의 `InfoBanner`·`GradeChip`·`StatusBadge`·`DistributionBarChart`·`Tabs`·`PageHeader`·`Select`·`States(Empty/Loading)`를 그대로 쓰고, 신규는 **3개**만(`YoyTimelineChart`·`YoyDistributionGroup`·`LegalEntityFilter`).
> 신규 토큰 = 0. 등급색은 기존 `grade-s~d`/`gradeBg-*`/`gradeFg-*` 재사용. 차트는 라이브러리 없는 **인라인 SVG**(기존 `MonthlyTrendChart` 패턴).

## 0. 진입 동선 / 메뉴 위치

주 사용자 = **hr_admin · division_head**. employee/team_lead는 본인·소속 범위에서 제한 노출(RBAC `visibilityScope` 준수).

```
사이드바  [모니터링] 그룹
  ├ 이의제기            /appeals
  └ 연도 비교 ★신규     /reports/yoy        tone: 'eval'  아이콘: TrendingUp
```

- `lib/nav.ts` NAV_ITEMS에 1개 항목 추가(`key: 'yoy'`, `group: '모니터링'`, `roles: ['hr_admin','division_head','team_lead']`). `activeKeyForPath`에 `/reports/yoy → 'yoy'` (`/reports`보다 **먼저** 매칭).
- 보조 진입(딥링크):
  - **평가결과 상세**(`/eval/result`) 헤더 우측 → `[연도 비교 ▸]` 버튼 → `/reports/yoy?tab=person&userId=<당사자>`.
  - **분포 모니터링**(`/reports`) 그룹 카드 → `[연도별 비교]` 링크 → `/reports/yoy?tab=org&scope=group&orgId=<id>`.
  - **대시보드**(hr_admin) 위젯 "연도 비교 바로가기".

라우트: `/reports/yoy` (탭 쿼리 `?tab=person|org`). 두 화면을 한 페이지의 탭으로 묶는다(개인 타임라인 / 조직 등급분포).

### 2.2 동작 / 상태

- **PersonPicker**: 검색형 Select. 결과/딥링크로 `userId` 들어오면 프리셀렉트. 퇴사자는 목록 하단 "퇴사자 포함" 토글 ON일 때만 노출 + `[퇴사]` 뱃지.
- **퇴사자 헤더 뱃지**: 이름 옆 `StatusBadge`(중립 톤) "퇴사" + 회색 처리. 카드/차트는 정상 렌더(과거 데이터 보존).
- **사이클 멀티셀렉트**: 닫힌(closed) 사이클만. 기본 = 해당 인물 결과가 있는 전 사이클. 2개 미만이면 차트 대신 단일 카드.
- **규칙 미반영 연도**: 역량 점수는 "82 (참고)" 캡션. 등급 추이선에서 점수 비반영 사이클은 회색 점 + 점선.
- 빈 상태: 결과 없음 → `States.Empty` "이 임직원의 과거 평가 결과가 아직 없어요."
- 로딩: `States.Loading` 스켈레톤(차트 영역 + 카드 3개).
- RBAC: division_head는 본인 본부 인원만 PersonPicker에 노출(백엔드 스코핑). 권한 외 userId 딥링크 → 403 빈 상태("열람 권한이 없는 임직원이에요").

### 2.3 반응형

| 구간 | 동작 |
|------|------|
| lg ≥1024 | 차트 풀폭 + 연도 카드 3열 그리드 |
| md 768–1023 | 차트 풀폭 + 연도 카드 2열, 사이클>2면 가로 스크롤 |
| < 768 | 차트 `overflow-x-auto`(min-w 480) + 연도 카드 1열 세로 스택. 사람 선택바 세로 쌓기 |

### 3.2 동작 / 상태

- **단위 토글**(그룹/본부/팀): 세그먼트 버튼(`OrgViewToggle` 패턴 재사용 가능, 신규 스타일 없음). 선택 단위에 맞춰 OrgPicker 옵션 갱신.
- **OrgPicker**: 통합 조직 트리(법인 무관). 법인필터는 **막대/테이블의 인원 집계에만** 적용(소속 인원 중 해당 법인만 카운트) — 트리 구조는 유지.
- **법인필터 = 전체 / 에너지엑스㈜ / 미래환경플랜**: 분포 카운트를 필터링. "전체"가 기본. 미래환경플랜은 여러 그룹에 걸치므로 그룹 단위에서도 의미 있음.
- **퇴사자 토글**: ON이면 과거 분포에 퇴사자 포함(당시 재직 기준). OFF면 현재 재직자만. 기본 OFF는 "당시 기준" 분포 왜곡 → **기본 ON**(과거 분포는 당시 인원이 맞음) + 캡션으로 명시.
- 단일 사이클만 선택 → 막대 1줄 + 테이블 1행(비교 아님, 경고 아님).
- 빈 상태: 해당 조직 결과 없음 → `States.Empty`.
- RBAC: division_head는 본인 본부/하위 팀만 OrgPicker 노출. team_lead는 본인 팀.

### 3.3 반응형

| 구간 | 동작 |
|------|------|
| lg ≥1024 | 막대 그룹 + 테이블 풀폭 |
| md | 막대 그룹(높이 축소) + 테이블 가로 스크롤(`overflow-x-auto`) |
| < 768 | 막대 세로 스택(연도별 카드) + 테이블 → 연도별 카드 리스트(등급 행) |

## 4. 공유 컴포넌트 배치도

```
페이지(YoyComparePage)
 ├ PageHeader (재사용)  ├─ right: LegalEntityFilter(신규) + 퇴사자토글(신규/소형)
 ├ Tabs (재사용)
 ├ [person] PersonTimelinePanel
 │    ├ PersonPicker(재사용 Select+검색)  · 퇴사 StatusBadge(재사용)
 │    ├ CycleMultiSelect (신규-소형 칩) 
 │    ├ YoyTimelineChart (신규, SVG)
 │    ├ YearDetailCard ×N (신규-소형: GradeChip + RuleSetChip 재사용/소형)

 └ [org] OrgDistributionPanel
      ├ 단위토글(OrgViewToggle 패턴) · OrgPicker(재사용 Select)
      ├ CycleMultiSelect (공유)
```

## 5. 데이터 의존(백엔드 계약 — requirements §S4)

- 개인: `GET /api/v1/results/compare?userId=&cycleIds=` → `{ user, cycles:[{ cycleId, year, finalGrade, finalScore, perfScore, compScore?, orgSnapshot:{group,division,team}, ruleSummary:{competencyIncluded, competencyReflected, perfWeight} }] }`.
- 조직: `GET /api/v1/results/distribution?scope=&orgId=&cycleIds=&legalEntity=&includeResigned=` → `{ scope, org, cycles:[{ year, total, counts:{S,A,B,C,D}, avgGrade?, poolCaps?:{S..D} }] }`.
- 정규화: 백엔드가 S~D·100점 공통축 + `ruleSummary`로 규칙차이 표면화. 프론트는 표시만(재계산 금지).
- 응답 봉투 unwrap·camelCase는 기존 훅 패턴(`useEvaluations` 등) 준수.
